🔧 PDFボタン表示修正のデモ

問題の概要

PDFのみのドキュメント（URLがない）の場合、PDFダウンロードボタンが表示されない問題がありました。

これは document_card.html テンプレートの条件分岐で、PDFボタンが document.url の条件の中にネストされていたためです。

修正内容

修正前の構造 ❌

{% if document.domain %} {% if document.url %} 元記事 {% if document.pdf_path %} PDF ← URLがないと表示されない！ {% endif %} {% endif %} {% endif %}

修正後の構造 ✅

{% if document.domain %} ドメイン {% endif %} {% if document.url %} 元記事 {% endif %} {% if document.pdf_path %} PDF ← 独立して表示される！ {% endif %}

動作確認

ケース1: PDFのみ（URLなし）

修正前 ❌

手動 example.com
❌ PDFボタンが表示されない

修正後 ✅

手動 example.com 📄 PDF 詳細を見る
✅ PDFボタンが正しく表示される

ケース2: URLとPDF両方あり

修正前 ✅

手動 example.com 🔗 元記事 📄 PDF 詳細を見る
✅ 両方のボタンが表示される

修正後 ✅

手動 example.com 🔗 元記事 📄 PDF 詳細を見る
✅ 両方のボタンが表示される（変更なし）

ケース3: URLのみ（PDFなし）

修正前 ✅

手動 example.com 🔗 元記事 詳細を見る
✅ URLボタンのみ表示される

修正後 ✅

手動 example.com 🔗 元記事 詳細を見る
✅ URLボタンのみ表示される（変更なし）

テスト結果

  • ✅ Test 1: PDFのみ（URLなし）の場合 - PDFボタンが表示される
  • ✅ Test 2: URLとPDF両方ある場合 - 両方のボタンが表示される
  • ✅ Test 3: URLのみ（PDFなし）の場合 - URLボタンのみが表示される
  • ✅ Test 4: URLもPDFもない場合 - どちらのボタンも表示されない

影響範囲

  • 変更ファイル: app/templates/partials/document_card.html
  • 影響を受けるページ: ドキュメント一覧ページ、ブックマークページ
  • 後方互換性: 既存の動作を保持しつつ、PDFのみのケースで正しく動作するように改善
